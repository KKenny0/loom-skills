# Phase 1: Capture

保存原始素材，识别结构，为精读做准备。

## Gate: 素材检查

**在执行任何后续步骤之前，检查用户是否提供了至少一个素材（URL / PDF / 文件 / 粘贴文本）。**

如果没有任何素材：
- **停下来**，向用户说明 deep-read 需要原始素材作为分析基础
- 请用户提供具体的 URL、文件、或直接粘贴文本
- 不要自行搜索主题、不要用通用知识写文章
- 这不是可选建议——没有素材就不启动流水线

为什么这很重要：deep-read 的整个价值链建立在「有据可查」上。每个论断都需要素材证据支撑。没有素材的 deep-read 输出只是通用知识的重新包装，不是研究文章。

## Step 1: 获取素材

对每个素材来源，根据类型获取内容：

| 来源类型 | 获取方式 |
|----------|----------|
| URL | `mcp__web-reader__webReader` 抓取全文 |
| PDF 文件 | `Read` 工具直接读取 |
| 本地文件 | `Read` 工具直接读取 |
| 粘贴文本 | 直接使用用户提供的文本 |

抓取规则：
- 保留原文用词和结构，不做 AI 评论或修辞加工
- 如果 URL 无法访问（404、付费墙），尝试 web cache；仍然失败则告诉用户，不编造内容
- 非中文素材保留原文关键术语，笔记用中文写

## Step 2: 识别类型和结构

对每个素材识别：

1. **素材类型**：论文、文章、访谈、报告、政策文件、书籍章节等
2. **语言**：中文/英文/其他
3. **结构元素**：标题层级、列表、表格、引用、图表描述
4. **长度**：短（<3000 字）、中（3000-10000 字）、长（>10000 字）

根据类型和长度，决定精读策略：
- 短素材：主线程直接精读
- 中/长素材：启动 background agent 并行精读

## Step 3: 推断研究主题

如果用户没有提供 `topic`，从素材内容推断：
- 所有素材共同指向的主题
- 用一个简洁的短语概括（2-6 字）
- 如果素材之间无明显关联，分别处理，向用户确认主题

## Step 4: 准备精读

完成 Capture 后，进入 [Phase 2: Read](02-read.md)。

如果有多个中/长素材，准备并行精读：每个素材一个 background agent，在一条消息中同时启动所有 agent。

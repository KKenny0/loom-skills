# Phase 3: Synthesize

综合分析所有 Source Brief，多视角压力测试，产出 Synthesis Pack。

## Step 1: 七步分解（内部推理）

对 Source Briefs 中的所有发现执行七步分解，只在内部推理，不输出逐步过程：

1. **铺现象**：列出所有素材中观察到的现象
2. **列候选**：从现象中提取可能的生成因素（generator）
3. **递归追问**：对每个候选因素追问"为什么它能生成这些现象？"
4. **合并同源**：把追问到底指向同一根源的候选合并
5. **砍**：去掉无法独立解释任何现象的候选
6. **反生成**：用剩下的候选反推，看能否重新生成观察到的现象
7. **预测+变更双测**：如果这个候选是对的，它还预测什么？什么条件下它会失效？

对每个存活候选，内部评估 5 个标准：独立性、必要性、生成力、简洁性、可证伪性。

然后写一段简短的 **Analysis Summary**（3-5 句）：哪些候选存活、哪个标准最弱。

## Step 2: 真实人物辩论（内部推理）

### 选人

邀请 3-5 位真实历史/当代人物参与辩论。

选人原则：
- 每位人物必须跟主题有真实的学术/思想关系
- 每位人物有明确的立场（一句话）
- 人物之间形成张力网络（非简单正反方）
- 至少一位"意外视角"——来自主题本身领域之外
- 有经典著作或知名言论的人物优先

### 辩论规则

每轮辩论：
- 每位人物必须引用 Source Brief 中的具体证据
- 没有素材支撑的观点不参与
- 每位人物回应证据和其他人物的观点

### 主持人原则

- **挖深不铺广**：每轮只追一条最深的裂缝
- **求真 > 和谐**：不追求表面共识
- 综合时暴露讨论的**结构**（假设、前提、推理链），不只复述内容

### 输出

写一段简短的 **Stress Test Summary**（3-5 句）：哪些视角趋同、在哪里分歧、证据最支持什么。只在 ASCII 框架图比文字更清晰时才画图。

## Step 3: 产出 Synthesis Pack

```
## Synthesis Pack

### Key Findings
- {发现 1}
- {发现 2}
- {发现 3}
- ...

### Working Thesis
{一句话：综合分析后，证据最支持的结论}

### Analysis Summary
{3-5 句：哪些 generator 存活、哪个标准最弱}

### Stress Test Summary
{3-5 句：哪些视角趋同、分歧在哪、证据最支持什么}

### Merged Conclusions
- {结论 1}: {证据支撑}
- {结论 2}: {证据支撑}
- ...

### Conflicts
- {冲突点}: {哪些视角/素材之间矛盾，证据分别是什么}

### Evidence Weight
- {论断 A}: 强支撑 / 中等 / 弱 / 争议
- {论断 B}: ...

### High-Risk Claims
- {高风险论断}: {为什么风险高，什么条件下会失效}
```

Synthesize 规则：
- 不隐藏未解决的冲突，不制造虚假共识
- 区分素材共识和用户论点
- 高风险论断必须标记
- Synthesis Pack 是只读的，Compose 阶段可以读但不能改

## Step 4: 完成 Synthesize 阶段

Synthesis Pack 完成后，进入 [Phase 4: Compose](04-compose.md)。

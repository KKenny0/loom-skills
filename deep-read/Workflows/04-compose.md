# Phase 4: Compose

从 Synthesis Pack 出发，撰写完整研究文章。内联三套规则：风格、去 AI 味、博导审稿。

---

## 内联规则 A：文章风格（来自 ljg-paper + ljg-writes）

### 核心原则

1. **一个锚点撑全文** — 全文用一个具体例子贯穿。读者一直待在同一个问题域里。换锚点 = 换地图 = 读者前面建立的直觉全丢。
2. **变形替代定义** — 解释两个概念关系时，把 A 连续变形成 B，不要说"A 和 B 是 XX 关系"。
3. **推理外显** — 模拟想明白的过程，不是呈现想明白的结果。让读者觉得结论差一步就是自己想到的。
4. **落点在能用** — 给出"这意味着你可以 ___"，而非"这让我们重新思考 ___"。
5. **外科医生的手，朋友的口** — 下刀冷静精准，说话直接不绕。

### 写作手法工具箱

选用，不必全用：

- **场景代替论证** — 不说"这是错的"，造一个场景让读者自己看到它错
- **让步弯道** — 最强势判断之后踩一脚刹车。承认对面有道理，然后再断言
- **反问入链** — 遇到隐含前提，用问题打开，然后回答
- **探索性语气** — "X 看起来是一回事，但如果你……等等，这意味着 Y"
- **短句做锤子** — "就这样。""没了。"全文最多两三处

### 5-step 文章结构（论证型素材）

1. **摆观点** — 把核心观点放到台面上，一句话，不模糊不铺垫
2. **切第一刀** — 反问/追问/翻转，切出读者没看见的层
3. **切第二刀** — 更抽象的层，用一个具体画面扳回来
4. **切到底** — 到切不动为止。切不动 = 挖到不可再分的事实，或诚实的"不确定"
5. **综合** — 从底回看第一步那句话，说清楚它怎么变了。结尾不总结，最后一句是最后一个发现或一扇门

### 叙事型素材

如果素材不是论证型（访谈、报告、新闻调查），不用 5-step 结构，用更自由的叙事结构，但仍然遵守核心原则 1-5。

### 语言规则

- 短词优先："进行分析"→"看"
- 一句一事：每句只推进一步
- 具体优先：名词看得见，动词有力气，形容词能砍就砍
- 不填充：删开场白、拐杖词、夸大象征
- 信任读者：说一遍够了，不重复结论
- 破公式：否定式排比全文不超过两处
- 变节奏：长短句交替
- 杀金句：听起来像可引用的，重写

## 内联规则 B：去 AI 味

### 最高优先级：保语义 > 去 AI 味

先确认事实、逻辑、因果不变，再做以下改写。如果为"更口语"改坏原意，属于失败改写。

### 10 条 AI 痕迹（写完每章后立即检查）

1. **段末收尾总结句**："到这里"、"这说明"、"这本身就是"、"可以看出"开头的短抽象句。段落最后一句比其他句短且抽象。直接删。
2. **升华句**：把具体观察上升到普适人生道理。直接给具体建议，不升华。
3. **对比句式**："不是…而是…"、"X 本身没有价值，真正有用的是 Y"。直接说结论，不用对比框架铺垫。
4. **bold 小标题模式**：`**xxx**。content` 改为 `**xxx**，content`，让 bold 变成承重词，不是独立小标题。例外：bold 部分本身是完整句子时保留句号。
5. **章节引介过渡句**：章节开头或结尾写"上面这些模式解决的是…，下面再看…"。直接删掉，章节标题本身已经承接。
6. **空泛形容词预判**："更干净："、"逻辑很清晰："，形容词抢先下判断。删掉形容词，只留后面的事实。
7. **工整并列 bold 标题**：四个 bold 标题全是同一格式。每个点语气不同，或改成散文叙述。
8. **段内重复**：同一段把同一意思用不同措辞说两遍。说一遍。
9. **讲解腔起手**："真拆开看"、"这背后是同一个变化"、"真正关键的问题是"。删掉，直接说。
10. **训人感起手**："先问问是不是"、"你要先明白"。改成"可以先看"、"先确认"。

### 句式规则

- 禁止用破折号（—），用逗号或分号代替
- 四五个句号连发、每句很短 = 打电报，合成长句或用逗号连
- 单独成段的一两句话，多半是上一段的收尾，直接并进去
- 不用"首先…其次…最后"

### 用词去正式化

| 不用 | 用 |
|------|-----|
| 非常/极其 | 很 |
| 综上所述 | 直接收尾 |
| 例如 | 比如 |
| 值得注意的是 | 直接说结论 |

## 内联规则 C：博导审稿

换身份：这个方向带了二十年研究生的博导，判断文章质量。

### 审查维度

- **论证力度**：每个论断有据可查吗？论据来源可靠吗？
- **预设审计**：文章最根本的预设是什么？这个预设站得住吗？如果这个预设塌了，文章还在不在？
- **逻辑链**：推理过程有没有跳跃？有没有用结论证明结论？
- **反直觉发现**：文章有没有一个"只有深入分析才能发现"的反直觉结论？没有 → 分析不够深
- **诚实度**：有没有把不确定性藏起来？有没有强做结论的地方？

### 审查结果

逐条给出修改建议。回去改。改完重新过。直到博导通过。

## Step 1: 确认锚点

从 Synthesis Pack 的结果中，选定一个具象的锚点（具体场景/例子），贯穿全文。

原则：
- 一个锚点撑全文，不换锚点
- 锚点要简单到一两句话能说清
- 如果同一锚点无法支撑某些章节，引入"子锚点"（必须跟主锚点在同一问题域）

## Step 2: 规划章节

基于 Synthesis Pack 的结论和发现，规划最终章节结构：

- 每章标题简洁（≤ 10 字）
- 章节之间有逻辑依赖：前面章节为后面铺垫
- 6-8 章为佳

向用户展示章节规划，等确认后开始写作。

## Step 3: 逐章写作

按章节顺序写作，每章流程：

### 3.1 回顾素材

从 Read 阶段的 Source Brief 和 Synthesize 阶段的 Synthesis Pack 中找出与本章节相关的内容。

### 3.2 撰写章节

遵守 Rule A（文章风格规则）。

### 3.3 章节内审查

写完每章后立即检查：
- 口语检验：能大声读出来吗？
- AI 痕迹：10 条 AI 痕迹逐条过（Rule B）
- 锚点一致性：还在同一个例子上吗？

## Step 4: 全文审查

所有章节写完后，做两层审查：

### Rule B — 去 AI 味审查（全文扫描）

逐条检查：

- [ ] 没有段末收尾总结句
- [ ] 没有升华句
- [ ] 没有对比句式框架（"不是…而是…"）
- [ ] bold 是承重词不是独立小标题
- [ ] 没有章节引介过渡句
- [ ] 没有空泛形容词预判
- [ ] bold 标题语气各异
- [ ] 没有段内重复
- [ ] 没有讲解腔起手
- [ ] 没有训人感起手
- [ ] 没有破折号
- [ ] 没有打电报式短句连发
- [ ] 没有"首先…其次…最后"
- [ ] 用词已去正式化

任何一条不过，回去改。改完重新过。

### Rule C — 博导审稿

换身份做博导审稿（审查维度见上方 Rule C）。

审查结果：逐条给出修改建议，回去改，改完重新过。直到博导通过。

## Step 5: 写入文件 + 交付报告

获取时间戳：`date +%Y-%m-%d`

输出路径和文件开头格式见 [SKILL.md 的输出规范](../SKILL.md)。如果用户指定了 `output_dir`，优先使用用户路径。

用 Write 工具写入完整文件。写入后用 Read 工具确认文件已正确保存。

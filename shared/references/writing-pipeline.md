# Loom Research Writing Pipeline

This reference defines the source-preserving and voice-preserving workflow used by the Loom skill pack.

## Pipeline

```text
Input
→ Material List
→ Raw Capture + inbox Daily Note
→ Source Brief
→ Synthesis Pack
→ Draft / Final
→ Topic Note
→ Index / Connection / Evolution
```

## Stage Boundaries

### 1. Intake

Classify the task:

- single-source deep dive
- multi-source synthesis
- theme research
- existing draft rewrite

Create a Material List before reading or drafting.

### 2. Capture

Preserve source material and create an inbox Daily Note entry.

- Raw Capture stores the source or a faithful conversion.
- Daily Note points to the source and keeps minimal source-facing notes.
- Do not publish-polish captured material.

### 3. Read

Create one Source Brief per core source.

Read the full source when it is central to the argument. Do not rely only on summaries.

### 4. Synthesis Pack

Compare sources:

- merge conclusions
- compare conflicts
- weight evidence
- identify consensus and disagreement
- extract article thesis
- mark high-risk claims
- create Source Coverage Checklist

### 5. Draft / Final

Write article prose from the Synthesis Pack.

- Draft: content-complete version for feedback.
- Final: publish-ready version.

Draft/Final can use the user's voice, narrative, and rhetoric. They must not rewrite upstream research artifacts.

### 6. Topic / Index / Evolution

Convert trusted research into durable knowledge.

- Topic Notes use calm knowledge-base voice.
- Indexes and connections are structured metadata.
- Evolution summaries tell the story of knowledge development.

## Voice-Preserving Rules

- Raw Capture preserves source wording and source meaning.
- Daily Note preserves source semantics and avoids publication-style rewriting.
- Source Brief distinguishes source claims from evidence, limits, and claims to verify.
- Synthesis Pack distinguishes merged conclusions, conflicts, user thesis, and high-risk claims.
- Draft/Final may polish, reorganize, and strengthen voice.
- Draft/Final must not overwrite Raw Capture, Daily Note, Source Brief, Synthesis Pack, or Topic Note.
- Topic Note removes article rhetoric and returns to structure, mechanisms, boundaries, and reusable insights.

## Review Semantics

`reviewed` means facts, sources, user intent, and expression boundaries have been checked.

Topic Note input priority:

1. Synthesis Pack
2. `reviewed` Daily Notes
3. Final or Published Article

Do not use unreviewed Drafts for Topic Notes by default.

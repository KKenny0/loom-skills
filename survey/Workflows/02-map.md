# Phase 2: 测绘

将 Phase 1 的素材目录结构化为领域地图。五个理论模块，每个模块有明确的操作步骤。

这一阶段的核心工作不是"读更多素材"，而是"把已有素材放到正确的结构位置上"。

## 模块一：研究纲领（Lakatos）

理论基础：Lakatos 的科学研究纲领。一个领域通常同时存在多个研究纲领，每个纲领有自己的硬核（不可动摇的基本承诺）和正面启发法（指导追随者下一步该研究什么）。

### 步骤

1. **分组**：将素材目录中的素材按核心承诺分组。问自己：这些工作在回答同一个基本问题吗？它们共享哪些不可动摇的前提？

2. **识别纲领**：每组素材构成一个研究纲领。通常一个领域有 2-5 个纲领。对每个纲领：

   **硬核**：这个纲领认为什么是不可质疑的？它把什么当作给定的？
   - 硬核不是论文里写出来的——它是论文不需要论证就假设成立的东西
   - 从多篇论文的共享前提中推断，不要从单篇论文的声明中提取

   **保护带**：围绕硬核的可调整假设。当遇到反例时，保护带会先调整，而不是硬核。
   - 如果一个工作不断修改辅助假设来应对新证据，这些辅助假设就是保护带

   **代表工作**：正面启发法的产物——纲领鼓励追随者去做的工作。标注 2-3 个最重要的。

   **状态评估**：进步还是退化？
   - 进步纲领：不断产生新的预测和发现，引领领域向前
   - 退化纲领：主要工作是在打补丁、解释反例，而不是产生新洞察
   - 如果拿不准，标"中性"——不要为了下判断而勉强

3. **纲领间关系**：
   - 互补：纲领解决不同层面的问题，可以共存
   - 竞争：纲领对同一问题给出不同答案，互相排斥
   - 包含：一个纲领是另一个纲领的特例
   - 演化：一个纲领从另一个纲领中分化出来

### 输出

```
纲领 A: {名称}
  硬核: {一句话}
  代表工作: {S1, S3}
  状态: 进步
  正在探索: {当前方向}

纲领 B: {名称}
  硬核: {一句话}
  代表工作: {S5, S8}
  状态: 退化倾向
  保护带调整: {最近在打什么补丁}

关系: A 和 B 竞争（对 {问题} 给出不同答案）
```

## 模块二：核心争议（IBIS）

理论基础：Rittel 的 IBIS（Issue-Based Information Systems）。IBIS 用三种元素映射争论：Issue（需要回答的问题）、Position（对问题的一种回答）、Argument（支持或反对某个立场的理由）。

### 步骤

1. **从纲领竞争关系出发**：模块一识别出的竞争纲领之间，争议点就是 Issue。通常核心争议只有 2-4 个。

2. **对每个争议提取 IBIS 结构**：

   **Issue**：把争议表达为一个问题。好的 Issue 是双方都必须回答的，而不是只有一方关心。
   - 坏例子："X 方法好不好"（太笼统）
   - 好例子："解决 Y 问题应该从底层机制出发还是从上层行为出发？"

   **Positions**：每个纲领对 Issue 的回答。用纲领自己的语言表述，不要用另一方的框架重述。

   **Arguments**：
   - 支持 Position 的论据：来自代表工作的证据
   - 反对 Position 的论据：来自竞争纲领或批评性工作的证据
   - 每个论据必须指向具体素材（用 Source Catalog ID），不能凭空

3. **评估争议状态**：

   | 状态 | 判断标准 |
   |------|---------|
   | 活跃 | 双方都在积极产出新证据，尚未出现明显优势方 |
   | 趋于共识 | 一方的证据积累明显超过另一方，但尚未被普遍接受 |
   | 僵持 | 双方证据相当，且立场基于不同的价值判断，难以通过证据解决 |
   | 已解决 | 共识已形成，但可能有少数坚持者 |

### 输出

```
争议 1: {Issue}

Position A（纲领 A 方向）:
  论据+: {S1 证明了...}, {S3 表明...}
  论据-: {S7 指出...}

Position B（纲领 B 方向）:
  论据+: {S5 展示了...}, {S8 发现...}
  论据-: {S2 质疑...}

状态: 活跃
```

## 模块三：演化脉络（Kuhn）

理论基础：Kuhn 的范式生命周期。领域不是线性进步的，而是经历常规科学 → 反常累积 → 危机 → 革命的相变过程。

### 步骤

1. **按时间排列关键工作和事件**：用素材目录中的发表时间建立粗略时间线。

2. **标记相变点**：不要只是列时间轴。识别领域从一个阶段转到另一个阶段的转折点：

   **常规科学期**的特征：
   - 大量工作在已有框架内解谜
   - 共识稳定，论文主要在"应用"而非"挑战"框架
   - 术语和问题定义趋于统一

   **反常累积期**的特征：
   - 越来越多"现有方法解决不了"的案例出现
   - 出现"为什么 X 不 work"类论文
   - 修补式工作增多（加约束、改假设、调参数）

   **危机期**的特征：
   - 出现"X 是否还有前途"的质疑
   - 研究者开始跳到其他框架寻找灵感
   - 领域的研讨会和综述中开始出现"反思"和"重新审视"

   **革命期**的特征：
   - 新框架出现，重新定义什么算"好问题"
   - 新术语大量涌现
   - 旧框架的工作开始用新框架的术语重新表述

3. **定位当前阶段**：领域现在处于哪个阶段？这决定了领域地图的基调——常规科学期的地图侧重共识，危机期的地图侧重张力。

### 输出

```
| 时期 | 阶段 | 标志性事件/工作 | 来源 |
|------|------|----------------|------|
| 20XX-20XX | 常规科学 | {纲领A的系列应用} | S1, S3 |
| 20XX-20XX | 反常累积 | {关键反例开始出现} | S7 |
| 20XX-20XX | 危机 | {反思论文出现} | S10 |
| 20XX-至今 | 革命 | {新框架出现} | S12, S15 |

当前阶段: {评估}
```

## 模块四：开放问题（Popper）

理论基础：Popper 的问题驱动认识论。科学从问题开始（P1），经过尝试性理论（TT）和排错（EE），产生新问题（P2）。领域的前沿由 P2 定义——上一个循环产生但尚未解决的问题。

### 步骤

1. **识别当前 P2 问题**：
   - 从前沿素材中提取研究者明确提到的 open problems、challenges、limitations
   - 从综述性素材的 "future work" 部分提取
   - 从争议性素材中提取双方都承认尚未解决的问题

2. **对每个开放问题描述**：

   **问题**：用一句话说清楚问题是什么。不要用行话包装。

   **谁在攻**：哪些研究团队或纲领正在尝试解决？用素材目录中的来源支撑。

   **什么路线**：他们用什么方法？是改进现有方法、引入新工具、还是重新定义问题？

   **卡在哪**：当前的主要瓶颈是什么？是数据不够、理论不完备、还是工程限制？

3. **识别被忽视的问题**：
   - 有些问题明显重要但没人关注——可能因为它们不属于任何现有纲领的核心关切
   - 也可能是跨纲领的交叉地带，每个纲领都觉得"那是别人的问题"
   - 这类问题最有发现价值，但也最不确定——标注 `[推测，需验证]`

### 输出

```
开放问题 1: {一句话描述}
  谁在攻: {团队/纲领} (S5, S8)
  路线: {方法}
  瓶颈: {卡在哪}

开放问题 2: {...}

被忽视的方向: {描述} [推测，需验证]
```

## 模块五：入口推荐（文献计量结构分析）

理论基础：引用网络的拓扑分析。不用做完整的引文分析——通过搜索结果中的引用关系、综述中的引用、和领域常识，就能识别出三种结构角色的文献。

### 步骤

1. **筛选 3-5 个入口**：

   | 优先级 | 角色 | 选择标准 | 读者获得什么 |
   |--------|------|---------|------------|
   | 1 | 奠基性 | 被广泛引用，定义了核心概念 | 术语和基本框架 |
   | 2 | 桥梁性 | 覆盖多条路线，给出全局对比 | 全局视角 |
   | 3-5 | 前沿性 | 近 1-2 年最活跃的方向 | 当前状态 |

2. **每个入口附推荐理由**：不要只说"这是一篇好论文"。说清楚：
   - 读完这篇之后，读者能理解什么之前不理解的东西
   - 它在领域地图中的位置（属于哪个纲领、处于哪个时期）
   - 读完之后应该接着读什么

3. **如果领域有好的综述**：优先推荐综述作为第一个入口——综述本身就是一份小型的领域地图，读完后用户能更有针对性地选择下一步。

### 输出

```
入门推荐:
1. {奠基性工作} (S1)
   类型: 奠基
   为什么先读: 定义了 {核心概念}，是理解后续工作的前提
   读完后: 建议接着读 {桥梁性工作} 了解全局

2. {桥梁性工作} (S6)
   类型: 桥梁
   为什么: 对比了 {纲领A} 和 {纲领B}，读完能建立全局视角
   读完后: 根据兴趣选择 {纲领A的前沿} 或 {纲领B的前沿}

3. {前沿工作} (S15)
   类型: 前沿
   为什么: 代表了 {最新方向}，是当前最活跃的研究线
   读完后: 建议用 deep-read 深读
```

## 完成标准

Phase 2 完成时，你应该有：

1. 2-5 个研究纲领的完整描述（硬核、代表工作、状态）
2. 2-4 个核心争议的 IBIS 映射
3. 标注了相变点的时间线
4. 3-5 个开放问题 + 1-2 个被忽视的方向
5. 3-5 个入门推荐，每个附推荐理由

所有结论都有素材目录中的来源支撑。没有来源支撑的标注 `[待验证]`。

这些结构化数据将直接用于 Phase 3 的撰述。

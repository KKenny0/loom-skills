# Phase 3: 撰述

将 Phase 2 的测绘结果写成学案体格式的领域地图。

## 学案体结构

学案体源自中国传统学术史（黄宗羲《明儒学案》），天然适合映射领域结构：

```
学案 = 研究纲领
案主 = 代表工作
传授 = 影响链
论学 = 核心辩论
未竟 = 开放问题
著作 = 入门推荐
```

## 完整模板

```markdown
# {主题} 领域测绘

## 总案

[一段话：这个领域的核心问题是什么、当前有哪几条主要路线、最值得关注的方向是什么。让读者在 30 秒内建立全局印象。]

## 研究纲领

### 纲领 A: {名称}

**硬核**: {一句话：这个纲领认为什么是不可质疑的}

**代表工作**:
- {S1}: {一句话说明为什么重要}
- {S3}: {一句话说明为什么重要}

**状态**: 进步 / 退化倾向 / 中性

**正在探索**: {正面启发法的当前方向}

### 纲领 B: {名称}

{同上结构}

### 纲领间关系

{互补/竞争/包含/演化关系的简要说明}

## 论争

**当前状态**: 活跃 / 趋于共识 / 僵持
```

## 写作过程

### 1. 规划章节

从 Phase 2 的测绘结果中提取各模块的关键结论。先写一个骨架：每个 section 的一两句话要点。确保骨架内部逻辑一致——纲领描述、争议、时间线、开放问题之间不能矛盾。

### 2. 逐章撰述

按模板顺序写。每章写作时的注意力分配：

- **总案**：最后写。因为总案需要对全局的判断，写完其他所有部分后才看得清全局。
- **研究纲领**：每条纲领独立写，确保用纲领自己的语言表述，不用竞争纲领的框架。
- **论争**：确保正反方都有具体证据支撑。如果某一方证据不足，标注 `[证据偏少]`。
- **源流**：重点在相变点，不在时间轴。每个相变点要说清楚"为什么这里是转折"。
- **未竟**：开放问题要有锋芒——不是泛泛的"需要更多研究"，而是具体到"Y 方法在 Z 条件下失败，没人知道为什么"。
- **入门**：每个推荐要回答"读完这个之后，读者能做什么之前做不了的事"。
- **覆盖度声明**：诚实、具体。"搜索覆盖了英文学术文献和中文技术博客，但未覆盖付费墙内的期刊论文和会议 proceedings。"

### 3. 质量审查

**结构审查**：
- 纲领之间有没有遗漏？是否覆盖了素材目录中大部分素材？
- 争议是否真的是"核心"争议，还是琐碎的技术分歧？
- 时间线有没有明显的空白期？

**证据审查**：
- 每个论断是否指向具体来源？
- 没有来源支撑的论断是否标注了 `[待验证]`？
- 有没有被素材覆盖度不足导致的方向性偏差？

**风格审查**：
- 思辨声音：直接、有判断、不 hedge
- deep-read 9 条红线：口语检验、无术语、短词、一念一句、具体、理由先行、不废话、信任读者、诚实
- 中文母语：自然的中文，不能有 AI 味

### 4. 输出

写完审查通过后，用 Write 工具输出文件：

- 文件名: `{主题}-survey_{YYYY-MM-DD}.md`
- 路径: 遵循 SKILL.md 中定义的三级 fallback 规则
- frontmatter: 遵循 SKILL.md 中定义的格式

## 完成标准

Phase 3 完成时，你应该产出了：

1. 一份完整的学案体领域地图文件
2. 覆盖了全部五个模块（纲领、争议、源流、未竟、入门）
3. 每个论断有来源支撑或 `[待验证]` 标注
4. 覆盖度声明诚实具体
5. 通过质量审查（结构、证据、风格）
